Чтобы создать новые мини-отчет, необходимо создать папку в этой директории
с именем отчета (например ТЭС).

В папке создать файлы с расширением sql содержащие обыкновенный TSQL-код.
Можно создавать вложенный папки, при экспорте структура папок выходных фалов
будет соответствовать структуре папок с sql-файлами.

При создании мини-отчета передаются следующие параметры:
	
	NumDataSource - номер источника данных открытой дороги
	NumRoad - номер дороги открытой дороги
	RoadName - название дороги открытой дороги
	DistrictName - название района открытой дороги
	
Эти параметры подставляются вместо конструкций вида:
	
	/*$ParaName*/Param-Value/*$*/
	
Например, если значение RoadName есть "Иркутск-Оса-Усть-Уда", то:

	SELECT '/*$RoadName*/1234/*$*/'
	
замениться на

	SELECT 'Иркутск-Оса-Усть-Уда'

Обратите внимание что внешние одинарные кавычки в скрипте сохранились, 
а старое значение "1234" полностью заменилось.

Указывать значение параметра не обязательно, т.е. следующая конструкция является допустимой:

	SELECT '/*$DistrictName*//*$*/'

Чтобы программа sqltotxt подключалсь к базе необходимо создать файл .credentials
и загнать в него параметры подключения. Пример содержимого:
#==== .credentials ====# << эту строку использовать не надо
UserID: sa
DataSource: .
InitialCatalog: RoadsDB_Direct
Password:
#==== .credentials ====# << эту строку использовать не надо
	
Исходники sqltotxt:
https://github.com/SevenLines/sqltotxt.git
